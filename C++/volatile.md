# 用途

用于修饰变量，防止一些可能出现的突变(由程序之外的因素决定),通常是硬件上

- 多线程共享的变量应该由volatile修饰，避免编译器优化将变量存入寄存器，导致不同线程从内存和寄存器两个地方取变量值，导致不可预料的错误。但通常使用互斥锁和原子操作

- 在硬件操作中，用volatile修饰变量，定义变量是一个不稳定的变量，通知编译器不要优化，每次都从内存中读值

# 常见问题

1、参数可以同时是const和volatile（如 只读的状态寄存器）

-  const指程序不应该去修改它

- volatile指它可能被意想不到地改变

2、指针可以是volatile的

当中断服务子程序修改一个指向一个 buffer的指针时

3、volatile修饰可能被某些编译器未知因素改变的值，避免编译器的优化

4、volatile修饰的值每次都应从内存中取出
